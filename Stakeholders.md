## Stakeholder Analysis for the Personal Expense Tracker

| Role | Key Concerns | Pain Points| Success Metrics |
|------| -------------|------------|-----------------|
|Primary user|Ease of use and data privacy|Manual entry takes too long in current apps.|Logging an expense takes < 10 seconds.|
|System Administrator|Data integrity and backups.|Potential loss of data if a local file is deleted.|Automated weekly backup success rate of 100%.|
|Software Developer |Code maintainability and modularity.|Hard-to-read spaghetti code in prototypes.|New features can be added without breaking old ones.|
|Security Auditor|Protection of sensitive financial data.|Plain-text storage of personal spending habits.|Zero unencrypted financial data at rest.|
|Financial Consultant|Accuracy of data for tax/planning.|Clients provide disorganized receipts.|100% accuracy in monthly PDF exports.|
|Product owner|User retention and project feasibility.| Users stop using the app after 3 days.|50% increase in daily active users over 1 month.|
