## Reflection:
Challenges in balancing needs:
The primary challenge faced during this process was balancing the User's need for extreme 
simplicity with the Security Auditor's and Administrator's requirements for robust protection
and data integrity. While a user wants "one-click" entry, security protocols like session timeouts
and password hashing add friction to the user experience.

Additionally, as an individual developer, balancing Scalability with Feasibility is difficult.
Designing a system that can handle thousands of users (as requested by a Product Owner) requires 
more complex infrastructure than a simple local app. 
I plan on addressing this by choosing a modular "Containerized" approach, 
allowing the system to grow without needing a full rewrite.
